Relations Between Two Objects

You create relations from a selected object to a target object.

For a given object, a relation is defined with a:
  • Name
  • Target object
  • Cardinality

A foreign key is added to the object in the BUILD model to materialize the links between the data of the object part of a relation.

A relation can be specified as Read-only or Read-write. A relation is unidirectional but you can specify two relations of opposite directions between two objects.

You can create 1 or n relations from an object A to:
  • a target object B
  • multiple target objects
  • the same object A, that is, a self-pointing relation
In BUILD, you can create relations in two ways:
  • By manually drawing a line linking objects
  • By using the Add New Relation button under the Relations tab of the side pane.
Default values are:
  • Cardinality = "n"
  • Read-only
  • Name
The default values can be modified.
The name of the object must obey the following rules:
  • Maximum length is 40 characters
  • The first character must be a letter or an underscore "_"
  • Subsequent characters must be alphanumeric or an underscore "_"
  • The name of the relation may not be the same as another relation name in the data model.
