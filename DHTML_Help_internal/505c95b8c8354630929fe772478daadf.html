Rules for Adding Objects Only - Foreign Keys and Mapping Table
In order for the names of properties, objects, and relations to be valid you need to respect the following rules:

Rule

Error Message

The maxmum length of a name is 40 characters

Please check the name. Names may not contain more than 40 characters.

The first character of a name must be a letter or an underscore “_”

Please check the name. The first character of a name must be alphabetic or “_”.

Subsequent characters must be letters, numbers or an underscore “_”

Note

In the case of spaces before the first character and/or after the last one, the case of whitespaces before the first character and/or after the last one, the spaces are removed suring the import process, for example ' toto ' becomes 'toto'.

Please check the name. The next characters of a name must be alphanumeric or “_”.

Names of objects, properties of a same object, and relations between the same two objects may not be duplicated in the file

You cannot use duplicate names. Property (or Object or Relation) [name] is a duplicate name.

An object that already exists in the target data model cannot be added to, that is imported to, this data model

You cannot import an Object that already exists in the Data Model. Object [name] already exists.
