Formats

You can use one of three formats to create a data model from an Excel sample:

  • Table using a mapping table - recommended
  • Table using foreign keys
  • Excel sheet using foreign keys - alternative, useful when exporting data from an SAP transactional system such as ERP, CRM, and so on.

About Cardinality of Relations in a Data Model

BUILD supports relations with a cardinality 1 and n , that is, 1-1 and 1-n

In the mapping table format, the cardinality is deduced from the data in the mapping table:

  • Cardinality will be n if some elements of the source Object of the Relation are mapped to multiple elements of the target Object.

  • Cardinality will be 1 if all elements of the source Object of the Relation are mapped to one element of the target Object.

In the foreign key format, the cardinality is deduced from the column that stores the foreign key. If the column of the foreign key is in the Source Object of the Relation, the cardinality will be 1. If the column of the foreign key is in the Target Object, the cardinality will be n.
