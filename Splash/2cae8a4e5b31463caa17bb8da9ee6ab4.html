Relationships Between Two Objects

You create relationships from a selected object to a target object.

For a given object, a relationship is defined with:
  • A Name
  • A Target object
  • Cardinality

A foreign key is added to the object in the data model to materialize the links between the data of the object part of a relationship.

A relationship can be specified as Read-only or Read-write. A relationship is unidirectional but you can specify two relationships of opposite directions between two objects.

You can create 1 or n relationships from an object A to:
  • A target object B
  • Multiple target objects
  • The same object A, that is, a self-pointing relationship
You can create relationships in two ways:
  • By manually drawing a line linking objects.
  • By using the Add New Relationship button under the relationships tab of the side pane.
Default values are:
  • Cardinality = n
  • Read-only
  • Name = DefaultName
  • Target object = Current Object
The default values can be modified.
The name of the object must follow these rules::
  • The maximum length is 40 characters.
  • The first character must be a letter or an underscore "_".
  • Subsequent characters must be alphanumeric or an underscore "_". No spaces are allowed.
  • The name of the relationship may not be the same as another relationship name in the data model.
