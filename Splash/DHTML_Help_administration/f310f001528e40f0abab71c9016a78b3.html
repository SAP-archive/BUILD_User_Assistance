Set Domain Access

BUILD Administrators can add domains and assign roles and permissions to each domain. Users from these domains can then be assigned roles to access BUILD.

Context

To add a new domain:

Procedure

  1. On the left sidebar, select Access. The Access List is displayed. On first access to the Admin Console, the default access level is set No Access.
  2. Select New Domain to add a new domain to BUILD and assign permissions. Before you can add domains for users, you must first add your own domain and give it Full Access. Then you can add more domains.
  3. Enter the domain name in the format <domainname>.com, for example sap.com.
  4. From the Access dropdown, select the BUILD access level. You can choose from the following access levels:

    Access Level

    Permissions

    No Access

    Users cannot access BUILD at all.

    Study Invitation Only

    Users can access a BUILD study by invitation only as a Guest user.

    Study or Project Invitation Only

    Users can access a BUILD study or project by invitation only. When usesr are invited to the project, they become Standard users so they can collaborate on the project.

    Guest With Study Invitation

    Users can sign in to BUILD as Guest users and access the Study by invitation or by URL. Users can also access studies from study invitations sent to them by other users.

    Guest With Study or Project Invitation

    Users can sign in to BUILD as a Guest users and access the Study by invitation or by URL. When users are invited to a project, they become a Standard user so they can collaborate on the project.

    Full Access

    Users have full access to all BUILD features
  5. Repeat these steps for all the domains you want to add.
